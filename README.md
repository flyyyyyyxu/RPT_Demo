一个面向小红书商家的AI内容策略与笔记生成Demo —— **让 0 运营经验的商家，也能持续产出「有买家感」的种草笔记。**

它不只是“帮用户写文案”，更核心的是帮助商家完成从**商品信息 → 内容策略 → 笔记预览**的一次内容决策。

## 项目背景

在小红书电商场景里，很多商家会遇到几个共性问题：

- 有商品，但不知道该怎么包装成适合平台的内容
- 会写商品信息，但不会提炼成能打动用户的卖点
- 想参考竞品，但缺少快速洞察和拆解能力
- 直接用 AI 生成文案，结果常常空泛、模板化、不像真实笔记

所以这个 Demo 想解决的，不是“生成效率”本身，而是**内容判断成本高**的问题。

## 核心价值

这个产品最核心的价值是：

**帮助商家把“拍脑袋发内容”，变成“有依据地做内容”。**

相比直接一键生成文案，我更希望这个产品能先帮用户想清楚：

- 这篇内容该打什么卖点
- 这篇内容该写给谁看
- 应该参考竞品的哪些点
- 最终生成的内容是否真的适合小红书语境

也就是说，它解决的不只是“写”，更是“怎么判断”。

## 功能亮点

### 1. 商品信息驱动内容起点
用户输入商品名称、描述等基础信息后，系统基于商品做内容参数预设，降低开始创作的门槛。

### 2. 内容策略可配置
不是黑盒一键生成，而是让用户参与卖点、人群、风格等策略选择，让结果更可控。

### 3. 竞品洞察辅助决策
即使没有竞品链接，也可以基于商品信息快速生成竞品洞察，帮助用户理解同行在讲什么、自己该怎么差异化表达。

### 4. 生成结果更贴近平台语境
内容预览会结合商品信息、策略参数和竞品洞察共同生成，而不是简单套模板，让结果更接近真实的小红书种草内容。

## 使用流程

1. 填写商品信息  
2. 选择内容策略  
3. 查看竞品洞察与内容预览  
4. 生成更适合小红书场景的笔记内容

## 技术实现

- **前端：** React 19 + Vite + Tailwind CSS 4
- **大模型接入：** 已完成真实大模型 API 接入，用于竞品洞察生成、内容策略分析和笔记内容生成，不再只是静态示例页面
- **内容策略实现：** 通过“商品信息识别 + 品类映射 + 策略配置”完成内容策略预设，让生成结果不只是随机出文案，而是先形成卖点、人群、风格等判断依据
- **生成方式：** 规则预设 + 大模型生成的混合方案
- **图片能力展示：** 已加入图片生成相关功能展示，用于模拟小红书内容创作中封面图/配图辅助生成场景，让 Demo 更接近真实创作链路
- **整体链路：**
  1. 输入商品信息
  2. 生成或配置内容策略
  3. 输出竞品洞察
  4. 生成笔记预览
  5. 展示图片生成能力

这样设计的目的，是避免大模型直接“黑盒写稿”，而是把内容生成拆成可理解、可配置、可展示的产品流程，更符合真实商家在小红书场景下的创作决策过程。

## 为什么不是纯 AI 一键生成

因为在这个场景里，用户真正缺的不是“写字能力”，而是**把商品翻译成内容的能力**。

所以这个 Demo 更强调：

- 用 AI 降低内容判断门槛
- 用策略配置提升结果可控性
- 用竞品洞察增强内容可信度

相比“生成一篇文案”，我更想展示的是：  
**AI 如何真正帮助商家完成一次内容决策。**
